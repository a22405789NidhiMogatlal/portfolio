1. Arquitetura MVT do Django

Desenho da Arquitetura

Componentes

  • Model: Representa os dados e a base de dados.
  • View: Trata da lógica e processa pedidos.
  • Template: Define o que o utilizador vê.

Fluxo de Informação

  1. O utilizador faz um pedido
  2. A View recebe o pedido
  3. A View consulta o Model
  4. O Model devolve os dados
  5. A View envia os dados para o Template
  6. O Template gera HTML
  7. A resposta é enviada ao utilizador

2. Modelação

Diagrama Entidade-Relação

models.py

class Morada(models.Model):
    rua = models.CharField(max_length=100)
    localidade = models.CharField(max_length=100)
    familia = models.OneToOneField(
        Familia,
        on_delete=models.CASCADE,
        related_name='morada'
    )

class Familia(models.Model):
    nome = models.CharField(max_length=100)

class Aluno(models.Model):
    nome = models.CharField(max_length=100)

Explicação

A aplicação utiliza várias entidades como Morada, Família e Aluno. A relação entre Morada e Família é de um para um (OneToOne), ou seja, cada família tem apenas uma morada associada.

Também podem existir relações muitos-para-muitos (N:N), como por exemplo entre Aluno e Disciplina.
